# docs/ — Context

Design decisions, implementation plans, and specifications produced during plugin development. Reference-only — never modify these files during normal skill execution.

## Structure

| Folder | Contents |
|--------|---------|
| `superpowers/` | Plans and specs produced using the superpowers agentic workflow (brainstorming → spec → plan → implementation) |

## Usage

Load specific files when you need to understand *why* a design decision was made or when implementing a feature that has a corresponding spec. Do not load entire docs/ tree into context.
